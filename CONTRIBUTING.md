# Contributing to Let Africa Build (LAB)

Thank you for your interest in contributing to Let Africa Build (LAB).

LAB operates a structured Bitcoin Layer 1 (L1) and Layer 2 (L2) developer pipeline designed to produce sustained open-source contributors and infrastructure-focused builders.

This document outlines how to contribute effectively within LAB repositories.

---

## Contribution Philosophy

LAB prioritizes:

- Technical correctness
- Clear documentation
- Security awareness
- Measurable open-source output
- Long-term contributor retention

All contributions should align with LAB’s focus on Bitcoin L1 and L2 infrastructure development.

---

## Who Can Contribute

Contributors may include:

- LAB cohort participants
- Contributor Sprint members
- Fellowship candidates
- External open-source collaborators

All contributors are expected to follow repository standards and the LAB Code of Conduct.

---

## How to Contribute

1. Fork the repository.
2. Create a new feature branch.
3. Make well-documented changes.
4. Ensure your code or documentation is clear and structured.
5. Submit a Pull Request (PR) with a concise explanation of:
   - What was changed
   - Why it was changed
   - Any testing performed

---

## Pull Request Guidelines

Pull Requests should:

- Be focused on a single improvement or fix.
- Include clear commit messages.
- Avoid unrelated formatting changes.
- Reference any related Issue where applicable.

Security-sensitive changes must include reasoning and testing details.

---

## Issue Reporting

To report an issue:

1. Open a new GitHub Issue.
2. Clearly describe:
   - The problem
   - Steps to reproduce (if applicable)
   - Suggested improvement (if available)

Well-documented issues improve review speed and technical quality.

---

## Contributor Pathway

LAB contributors may progress through the following structured pathway:

Screening → Bootcamp → Contributor Sprint → Fellowship → Maintainer Path

Sustained, high-quality contributions may qualify contributors for:

- Mentorship support
- Fellowship consideration
- Increased technical responsibility
- Recognition in transparency reports

---

## Code Standards

Where applicable:

- Follow repository-specific coding guidelines.
- Maintain clarity over cleverness.
- Prioritize readability and security.
- Ensure documentation accompanies technical changes.

---

## Transparency & Attribution

LAB tracks measurable open-source contribution as part of its program model.

All accepted contributions remain publicly visible and attributed through GitHub.

---

By contributing to LAB repositories, you agree to operate within a professional, technically rigorous, and collaborative environment aligned with Bitcoin infrastructure development.
